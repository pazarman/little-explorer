# Skill 06: STEM Scope & Sequence (ages 2–5)

Use this to (a) ensure every game has a real STEM objective, (b) plan new games against coverage gaps,
and (c) sequence difficulty tiers by child development, not by arbitrary counts.

## Domains, developmental ladder, and current coverage

| Domain | 2–3 | 3–4 | 4–5 | Covered by | Status |
|---|---|---|---|---|---|
| Number sense | subitize 1–3 | count 1–5 with 1:1 correspondence | count to 10 + numeral, "how many" | Counting Critters, Yum Count, Dragon Feed, Countdown, Numbers/balloons | **Strong** |
| Geometry / shapes | circle, square | + triangle, star, heart | match + name, trace | Pizza (shape mode), Magic Tracing | OK |
| Patterns / algebra | AB | ABC, AAB | extend + create | Star Patterns | **Thin** (one game) |
| Measurement / comparison | big / small | tall/short, more/less | order by size | Big & Small (size only) | **Gap**: length/height, more/less, ordering |
| Sorting / classification / data | by 1 attribute | by 2 attributes | simple "which has more" | Big & Small (size) | **Gap**: sort by color/kind/shape; compare sets |
| Spatial reasoning | in / on / under | here/there, simple fit | positional language | — | **Gap** (none) |
| Logic / cause–effect | tap → reaction | sequence / order | simple if–then | implicit only | **Gap**: explicit ordering/sequencing |
| Scientific thinking | name + observe | living/nonliving, weather/seasons | predict | Animal games, Body Match | **Gap** (weather/seasons on roadmap) |

## Sequencing rule

Within a concept, tiers follow the developmental ladder above — not arbitrary counts.
Example (counting): tier 0 = subitize 1–3, tier 1 = count 1–5 with one-to-one correspondence,
tier 2 = count to 10 with the written numeral.

## Highest-ROI gaps to build next (learning value first)

1. **Sort / classify by color, shape, or kind** (not just size) — core early math; today only size exists.
2. **Pattern depth** — extend and create patterns; more pattern types.
3. **Spatial / positional language** (in / on / under) — zero coverage.
4. **More / less and ordering** (compare sets, seriation) — zero coverage.
5. **Measurement** (tall/short, long/short).

## Required objective tag for every game

Each game (existing or new) should declare, in a comment by its level object:
`Domain · Concept · Age band · Success = (what the child can now do)`.
This makes coverage auditable and ties the rubric's Learning-efficacy axis to a concrete claim.
